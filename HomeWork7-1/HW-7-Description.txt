8 У проекті є наступні класи і структури:
- ProfileViewController - основний екран для виведення інформації про користувача
- UserModel - структура для зберігання даних користувача
- AdminModel - структура для зберігання даних адміністратора
- DataProviderService - клас, який має 2 функуії, в кожній формуються обʼєкт user і admin

ProfileViewController має:
- 2 кнокпи (IBAction), при натисканні на кожну відбувається звернення до DataProviderService для отримання даних (user і admin відповідно), але функції без параметрів.
- функцію updateUI(), в якій закоментовано вивід інформації на екран

Завдання:
- Додати у проект протокол ProfileData, який має загальні функції або змінні для опису користувача
(avatarName, firstName, lastName, eMail, address)
- Використовуючи розширення (extension) підключити протокол до моделей UserModel та AdminModel і додати реалізацію функції або змінних доданого протоколу
- Змінити код функції updateUI() у ProfileViewController, щоб ця функція приймала параметр змінної протоколу ProfileData та заповнити даними відповідні елементи інтерфейсу
- Змініти функції loadUserData() та loadAdminData() у класі DataProviderService таким чином,
щоб параметром функції був кложур (closure), який приймає параметром обʼєкт типу ProfileData
і після формування обʼєктів user та admin передати ці обʼєкти у відповідних функціях
- У ProfileViewController у функціях loadUserDataButtonAction та loadAdminDataButtonAction реалізувати closure, який в середині викликає функцію updateUI() і передає параметром ProfileData відповідний обїʼєкт

-> Таким чином, результатом має бути виведення даних користувача однією функцією updateUI через параметр протоколу ProfileData для різних типів користувачів (UserModel та AdminModel)
